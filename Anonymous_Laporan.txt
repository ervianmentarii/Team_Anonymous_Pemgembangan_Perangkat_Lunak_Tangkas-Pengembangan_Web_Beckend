Dokumentasi & Deskripsi Proyek


================================================================================
DESKRIPSI SINGKAT
================================================================================

SmartHealth adalah sebuah aplikasi web sistem informasi kesehatan berbasis
fullstack yang dirancang khusus untuk mendukung operasional klinik kardiologi
secara digital dan modern. Aplikasi ini memungkinkan pasien untuk mendaftar,
login, melihat daftar dokter beserta jadwal praktiknya, dan membuat janji temu
konsultasi secara langsung melalui halaman web tanpa perlu menelepon atau datang
langsung ke klinik. Di sisi lain, staf medis dan administrator klinik dapat
mengelola data pasien, memantau seluruh janji temu, serta memperbarui status
konsultasi melalui dashboard khusus staff yang terintegrasi dengan database
secara real-time. Sistem ini dibangun dengan arsitektur REST API yang memisahkan
frontend dan backend secara jelas sehingga setiap bagian dapat dikerjakan,
dikembangkan, dan dipelihara secara mandiri oleh masing-masing anggota tim.
SmartHealth memanfaatkan teknologi mutakhir seperti React, Express.js, Prisma
ORM, dan MySQL untuk menghadirkan pengalaman pengguna yang cepat, responsif, dan
aman bagi seluruh pengguna, baik pasien maupun tenaga medis. Proyek ini
merupakan hasil kolaborasi tim yang terdiri dari empat developer dengan
pembagian peran yang jelas, mulai dari desain antarmuka, pengembangan backend,
manajemen database, hingga pengujian dan integrasi sistem secara menyeluruh.

FITUR YANG TERSEDIA
================================================================================

FITUR PASIEN

[1]  Registrasi Akun
     Pasien baru dapat mendaftar dengan mengisi data diri lengkap (nama,
     NIK, jenis kelamin, tanggal lahir, alamat, nomor telepon, email,
     password). Akun otomatis dibuat di tabel auth_user dengan role PASIEN.

[2]  Login Pasien
     Login menggunakan email dan password. Tombol quick-fill akun demo
     tersedia untuk kemudahan pengujian (5 akun pasien siap pakai).

[3]  Dashboard Pasien
     Menampilkan sambutan personal dan jadwal janji temu mendatang.

[4]  Lihat Daftar Dokter
     Penelusuran dokter dengan fitur pencarian nama / spesialisasi dan
     filter berdasarkan sub-spesialisasi kardiologi.

[5]  Detail & Profil Dokter
     Profil lengkap dokter: spesialisasi, pengalaman, rating, jadwal.

[6]  Jadwal Dokter
     Jadwal praktik per hari dengan jam buka, jam tutup, durasi slot.

[7]  Buat Janji Temu (Multi-Step)
     Step 1 - Pilih dokter (search + filter spesialisasi)
     Step 2 - Pilih tanggal (14 hari ke depan, visual kalender)
     Step 3 - Pilih slot waktu (otomatis dari jadwal dokter)
     Step 4 - Konfirmasi + isi keluhan/gejala

[8]  Riwayat Janji Temu
     Seluruh riwayat janji temu dengan status: Terjadwal / Selesai /
     Dibatalkan.

[9]  Rekam Medis
     Rekam medis dari kunjungan yang telah selesai, dengan catatan dokter.

[10] Layanan & Kontak
     Informasi layanan klinik dan cara menghubungi.

FITUR STAFF / ADMIN

[11] Login Staff
     Tab login terpisah untuk Admin dan Dokter. Validasi role otomatis,
     akun pasien ditolak masuk portal staff. Tersedia 6 akun demo
     (1 Admin + 5 Dokter) dengan tombol quick-fill.

[12] Dashboard Staff
     Statistik real-time: total janji hari ini, terjadwal, selesai,
     dibatalkan, dan total pasien terdaftar.

[13] Manajemen Janji Temu
     Lihat seluruh janji temu, filter status/tanggal, ubah status
     (Terjadwal -> Selesai / Dibatalkan).

[14] Data Pasien
     Daftar seluruh pasien terdaftar beserta informasi identitas & kontak.

CARA MENJALANKAN APLIKASI
================================================================================

Prasyarat:
- Node.js v18+
- MySQL 8.0 aktif (XAMPP / Laragon)
- Database web_uts sudah diimport dari file SQL

Terminal 1 - Backend:
  cd server
  npm install
  npx prisma generate
  node index.js
  -> Server berjalan di: http://localhost:5000

Terminal 2 - Frontend:
  cd frontend
  npm install
  npm run dev
  -> Buka browser: http://localhost:3000

Konfigurasi server/.env:
  DATABASE_URL="mysql://root:@127.0.0.1:3306/web_uts"
  JWT_SECRET="secret_key_pilihan_anda"
